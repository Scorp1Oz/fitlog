# fitlog 🏋️

Офлайн-трекер силових тренувань і бігу для Android. Усі дані зберігаються
локально в SQLite — без бекенду, акаунтів у хмарі чи мережевих залежностей у
рантаймі (єдині зовнішні виклики — безключові векторні тайли карти).

Темна + лаймова дизайн-система, brutalist-естетика, анімації на reanimated.

---

## Архітектура

```
┌──────────────────────────────────────────────────────────┐
│  UI (expo-router)                                          │
│  app/(tabs) ─ PagerView host  +  stack-екрани              │
│  screens/ ─ Home · Workout · Analytics · Run              │
├──────────────────────────────────────────────────────────┤
│  Рантайм-стан (Zustand, у пам'яті)                         │
│  workout-store (free|routine) · run-store (GPS-трек)      │
│  rest-settings · curtain · confirm                         │
├──────────────────────────────────────────────────────────┤
│  Доменна логіка (lib/)                                     │
│  streak · strength(1RM) · geo(haversine/pace) · reminders │
├──────────────────────────────────────────────────────────┤
│  Персистентність (db/)  expo-sqlite                        │
│  міграційний раннер (PRAGMA user_version) · per-profile   │
└──────────────────────────────────────────────────────────┘
        фонові підсистеми: expo-task-manager (GPS) ·
        expo-notifications (таймер відпочинку + нагадування)
```

- **Single source of truth** — SQLite. Усе агрегування (тоннаж, 1ПМ, стрік,
  історія) рахується SQL-запитами або чистими функціями над вибірками; кеш у
  React-стані тримається лише для рендера й перечитується по `useFocusEffect`.
- **Рантайм-стан** (активна сесія, активна пробіжка) живе в Zustand — він
  переживає навігацію між екранами, але не персиститься: завершена сесія
  пишеться в БД однією транзакцією, після чого стор скидається.
- **Мультипрофільність** наскрізна: майже кожна таблиця має `profile_id`, усі
  запити фільтрують по активному профілю.

## Модель даних

Схема — `migrations.ts`, версіонується через `PRAGMA user_version`. Правило
міграцій: **лише адитивні зміни** (нові таблиці/стовпці), щоб історія
користувача переживала оновлення. Поточна версія — **v6**.

| Таблиця | Призначення | Ключові поля |
|---|---|---|
| `profiles` | акаунти (локальні) | `username`, `password_hash/salt`, `avatar_uri`, `birthdate`, `active_program_id`, `demo_hidden` |
| `exercises` | бібліотека вправ + користувацькі | `name`, `name_uk`, `primary_muscle`, `equipment`, `category`, `is_custom`, `profile_id` |
| `routines` / `routine_exercises` | шаблони з цілями | `target_sets`, `rep_low`, `rep_high`, `position` |
| `programs` / `program_days` | розклад рутин по днях тижня | `weekday` (0=Пн…6=Нд) → `routine_id` |
| `sessions` / `session_sets` | виконані тренування | `weight`, `reps`, `set_index`, `completed`, `started_at`/`ended_at` |
| `runs` / `run_points` | пробіжки + GPS-трек | `distance_m`, `duration_s`, `avg_pace`; точки `seq`, `lat`, `lng`, `t` |

Похідні величини **не зберігаються**: тоннаж = `Σ(weight·reps)`, оцінка 1ПМ —
формула Еплі `w·(1 + reps/30)`, середній темп — `duration / (distance/1000)`.

## Ключові підсистеми

### Запис пробіжки (фон)
`expo-task-manager` реєструє фоновий таск локації (`run-task.ts`); оновлення
координат летять у `run-store`, який фільтрує GPS-шум (`<2 м` ігнорується) і
накопичує дистанцію через haversine. Трек пишеться навіть зі згорнутим
застосунком (foreground service). На стоп — лаймова завіса піднімається першою,
а вже під нею відбувається `stopRun → saveRun → navigate`, щоб не блимнути
порожнім екраном.

### Стрік (`lib/streak.ts`)
Чиста функція над множиною днів з активністю (`toDateKey`) і розкладом активної
програми:
- **без активної програми** — стрік тримається на будь-якій активності (вільне
  тренування або пробіжка); порожній день рве ланцюжок;
- **з активною програмою** — у пріоритеті розклад: дні відпочинку (де програма
  не призначила тренування) **не рвуть** стрік, а тренувальні треба «закрити»
  будь-якою активністю; сьогоднішній день живе до півночі.

Вогник у шапці (без числа) показує лише факт «стрік живий»; тап відкриває
календар-модалку, де `classifyDay` розмічає кожен день (активність / відпочинок
/ пропуск / сьогодні).

### Нагадування (`lib/reminders.ts`)
Локальні сповіщення зі стабільними `identifier` (повторне планування замінює
попереднє, тож не множаться):
- два умовні нагадування про стрік у дні, які «треба закрити»: **16:00–17:59**
  («час тренуватись») і **19:00–20:59** (пропозиція домашнього тренування);
  обидва знімаються, щойно день закрито;
- легкі щоденні нагадування про тренування та пробіжку у випадковий денний час.

### Переклад вправ
База `free-exercise-db` (873 вправи) англійською; українські назви — гібрид
словника й токенного рушія (`exercise-names.ts`), `name_uk` засівається при
встановленні та доганяється міграцією v4. Англійська `name` лишається — під
майбутній перемикач мови.

## Структура

```
src/
  app/          # маршрути expo-router: (tabs)/ пейджер-хост + stack-екрани
  screens/      # сторінки пейджера: Home / Workout / Analytics / Run
  auth/         # локальні акаунти: Context, екран входу, декор
  lock/         # екран-замок: біометрія, Context, «ворота»
  db/           # SQLite: схема, міграційний раннер, репозиторії, provider
  workout/      # активна сесія (Zustand), таймер відпочинку, useStartRoutine
  run/          # пробіжка: стор, фоновий GPS-таск, контроль, карта MapLibre
  exercises/    # переклад таксономії/назв, спільний пікер вправ
  profile/      # аватар, модалка профілю, хелпери
  theme/        # палітра-джерело (дублює токени global.css для SVG/нативу)
  components/   # таб-бар, ScreenTitle, календарі, шапки, завіса, свічіння,
                #   StreakModal, CheckPop
  lib/          # streak · strength · geo · date · responsive · notify · reminders
global.css      # дизайн-токени (@theme) + база Tailwind
assets/         # exercises.json (база вправ), іконки, splash
docs/           # план, дизайн-система, навчальні конспекти
```

## Розробка

```bash
npm install
npx expo start          # 'a' — Android, 'w' — веб, або QR в Expo Go
npx tsc --noEmit        # перевірка типів
npx eslint .            # лінт
```

Потрібен Node.js і **Expo Go** (Android, той самий Wi-Fi). Android Studio
обов'язковий лише для локальної збірки APK. Біометрія працює на пристрої з
налаштованим відбитком/обличчям; у браузері екран-замок не блокує (передбачено).

## Збірка APK

Тека `android/` згенерована (prebuild), тож офлайн-APK збирається локально через
Gradle:

```powershell
cd android
.\gradlew assembleRelease
```

Результат: `android\app\build\outputs\apk\release\app-release.apk` —
універсальний APK із вбудованим JS-бандлом.

> Іконка, splash і канали сповіщень — нативні ресурси: після їх зміни потрібен
> повний ребілд (або `npx expo prebuild -p android`), тоді як зміни в JS
> підхоплюються гарячим релоадом. Release підписується дебаг-ключем шаблону —
> годиться для особистого тесту/портфоліо, не для Play Store.
