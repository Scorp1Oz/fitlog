## 1. UX-дрібниці

### Перемикач «Залишитись у системі»
Керує, чи памʼятати вхід між запусками. Технічно — параметр `persist` у
`login/register`:
- `true` → зберігаємо `active_profile_id` у SecureStore (наступний запуск пускає
  одразу);
- `false` → навпаки, **прибираємо** збережений id (наступний запуск → екран входу).

### Запит біометрії після реєстрації
Щойно акаунт створено — `Alert.alert(...)` з «Так/Ні». «Так» вмикає замок
(`useLock().setEnabled(true)`). Показуємо **лише якщо** пристрій підтримує
біометрію. Це звʼязало шари Auth і Lock (екран входу бере `useLock()`, бо лежить
усередині `LockProvider`).

## 2. Візуальний шар — порядок шарів

Найважливіше — **порядок** (хто над ким малюється):

```
1. Радіальний градієнт (SVG)        ← найнижче
2. Анімовані хрестики (reanimated)
3. Матове скло (BlurView)           ← розмиває все, що НИЖЧЕ (1 і 2)
4. Форма (поля, кнопки)             ← найвище, чітке
```

Хрестики лежать **під склом** → стають мʼякими, «привидами». Якби поклали над
склом — були б чіткими.

### Радіальний градієнт (react-native-svg)
`expo-linear-gradient` уміє лише **лінійний**. Для радіального взяли
`react-native-svg`:
```tsx
<Svg style={StyleSheet.absoluteFill}>
  <Defs>
    <RadialGradient
      id="limeGlow"
      gradientUnits="userSpaceOnUse"
      cx={size.width * 0.5}
      cy={-size.height * 0.08}
      r={size.width * 0.95}
    >
      <Stop offset="0"   stopColor="#C8F135" stopOpacity={0.7} />
      <Stop offset="0.45" stopColor="#C8F135" stopOpacity={0.14} />
      <Stop offset="1"   stopColor="#0D0D0D" stopOpacity={0} />
    </RadialGradient>
  </Defs>
  <Rect width="100%" height="100%" fill="url(#limeGlow)" />
</Svg>
```

### Матове скло (expo-blur)
```tsx
<BlurView intensity={40} tint="dark" style={StyleSheet.absoluteFill} />
```

### Анімовані хрестики (react-native-reanimated)
Кожен хрестик — `Animated.Text "✕"` з власним циклом анімації на **UI-потоці**:
```tsx
const t = useSharedValue(0);
useEffect(() => {
  t.value = withDelay(delay,
    withRepeat(withTiming(1, { duration }), -1, true)); // 0↔1 нескінченно
}, []);
const style = useAnimatedStyle(() => ({
  transform: [
    { translateX: interpolate(t.value, [0,1], [0, driftX]) },
    { translateY: interpolate(t.value, [0,1], [0, driftY]) },
  ],
  opacity: interpolate(t.value, [0,1], [0.12, 0.4]),
}));
```
Розміри/позиції/швидкості — випадкові, рахуються один раз через `useMemo`.

> 🔑 Терміни: **useSharedValue**, **useAnimatedStyle**, **withRepeat/withTiming/
> withDelay**, **interpolate**, **UI thread vs JS thread**, **onLayout**.

## 3. ❗ ПОМИЛКИ, на які ми натрапили (і як полагодили)

### Помилка 1 — збірка падає на react-native-svg
```
Unable to resolve "./lib/extract/types" from
node_modules/react-native-svg/src/ReactNativeSVG.ts
```
**Причина:** я встановив `react-native-svg`, поки dev-сервер працював. Metro мав
**застарілий кеш** і не бачив нового модуля. Файл насправді існував, а моя окрема
збірка (`expo export`) проходила — це й довело, що проблема в кеші, не в коді.

**Фікс:** перезапустити сервер з очищенням кешу:
```
npx expo start -c
```
**Урок:** після `expo install` будь-якого **нативного** пакета Metro треба
перезапустити з `-c`. Звичайний `r` (reload) не допомагає — змінився список
модулів, а не лише JS.

### Помилка 2 — радіальний градієнт виходив еліпсом (стікав вниз-вліво)
**Причина:** SVG за замовчуванням використовує `gradientUnits="objectBoundingBox"`
— координати 0–1 відносно прямокутника. На **високому** екрані коло
розтягувалось вертикально в еліпс.

**Фікс:** `gradientUnits="userSpaceOnUse"` + задати `cx/cy/r` у **пікселях**.
Для цього виміряли екран через `onLayout`:
```tsx
const [size, setSize] = useState({ width: 0, height: 0 });
<View onLayout={(e) => setSize(e.nativeEvent.layout)} ... />
```
У пікселях радіус однаковий по обох осях → **ідеальне коло**.

### Помилка 3 — лайм «перемагав» на екрані
Світіння було завелике і яскраве. Підкрутили:
- центр у `cx 50% / cy -8%` (по центру, **за** верхнім краєм — самої точки центру
  не видно);
- радіус рахуємо від **ширини** (`width * 0.95`), а не від більшої сторони
  (раніше брали висоту — тому й розливалось);
- знизили `stopOpacity` (пік 0.7, швидке згасання) → темний домінує.

### Помилка 4 — перемикач виглядав темним
**Причина:** на Android трек `Switch` напівпрозорий, тож лайм здавався тьмяним.
**Фікс:** яскравий **повзунок** замість треку:
```tsx
trackColor={{ false: "#2A2A2A", true: "#6B7F2A" }}
thumbColor={stay ? "#C8F135" : "#E8E8E8"}
```

### Помилка 5 — хрестиків майже не було знизу
**Причина:** вони дрейфували **тільки вгору** (`driftY` завжди відʼємний), тож
нижня смуга оголювалась.
**Фікс:** дрейф у **обидва боки** (`driftY = (random-0.5)*50`) — хрестик
лишається біля місця появи, розподіл рівномірний. Заодно зменшили розмір і
додали кількість (14 → 19).

### Питання, що виникло — чи скидає кеш акаунти?
**Ні.** `expo start -c` чистить **кеш Metro** (на компʼютері, зібраний JS).
Акаунти лежать **на телефоні** — у базі `fitlog.db` (SQLite) і SecureStore. Це
різні речі: код окремо (компʼютер), дані окремо (телефон).
