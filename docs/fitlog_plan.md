# fitlog — План реалізації

## 1. Що це
Особистий офлайн-трекер силових тренувань для Android (APK).
Лише для себе: усі дані зберігаються на телефоні, без сервера й щомісячних витрат на хостинг.
Водночас — портфоліо-проєкт, що демонструє профільну роботу з даними (тоннаж, оцінка 1ПМ, теплова карта тіла, експорт CSV).

## 2. Технологічний стек
| Шар | Інструмент | Навіщо |
|---|---|---|
| Платформа | **Expo (React Native)** | той самий JS/JSX, що у вебі; APK без Android Studio |
| Стилі | **NativeWind** | Tailwind-класи, знайомі після веб-етапу |
| База даних | **expo-sqlite** | локальне сховище, єдине джерело правди, повний офлайн |
| Міграції | власний раннер на `PRAGMA user_version` | захист історії при оновленні APK |
| Стан UI | **Zustand** | активна сесія + таймер без зайвих ререндерів |
| Картинки | **expo-image** | нативне кешування, легкий APK |
| Графіки | **react-native-gifted-charts** | візуалізація прогресу |
| Сповіщення | **expo-notifications** | таймер відпочинку, нагадування |
| Дані вправ | **free-exercise-db** (~800, public domain) + укр. шар перекладу | бібліотека вправ із коробки |
| Збірка | **EAS Build** (`buildType: apk`) | готовий APK на телефон |

## 3. Архітектурні рішення (ухвалено на старті)
- **SQLite = джерело правди** для всього, що має пережити перезапуск.
- **Zustand = рантайм-стан**: активне тренування, таймер, фільтри. По завершенні сесії — скид у SQLite.
- **Міграції з v1**: версіоновані, лише додавання (ніколи не руйнівні), докочуються при старті.
- **Картинки** тягнуться за URL і кешуються expo-image — APK лишається легким.
- **Ключі API** (майбутній асистент) не зашиваємо в APK — лише через проксі-бекенд.

## 4. Схема БД — версія 1
```sql
CREATE TABLE exercises (
  id INTEGER PRIMARY KEY, name TEXT, name_uk TEXT,
  primary_muscle TEXT, equipment TEXT, category TEXT, level TEXT,
  images TEXT, is_custom INTEGER DEFAULT 0
);
CREATE TABLE routines (id INTEGER PRIMARY KEY, name TEXT, created_at INTEGER);
CREATE TABLE routine_exercises (
  routine_id INTEGER, exercise_id INTEGER, position INTEGER,
  target_sets INTEGER, rep_low INTEGER, rep_high INTEGER
);
CREATE TABLE sessions (
  id INTEGER PRIMARY KEY, routine_id INTEGER,
  started_at INTEGER, ended_at INTEGER, notes TEXT
);
CREATE TABLE session_sets (
  id INTEGER PRIMARY KEY, session_id INTEGER, exercise_id INTEGER,
  set_index INTEGER, weight REAL, reps INTEGER,
  type TEXT DEFAULT 'normal', completed INTEGER DEFAULT 1
);
```
Майбутні міграції (для прикладу): v2 — `ALTER TABLE session_sets ADD COLUMN rpe REAL;`; v3 — таблиці `bodyweight`, `measurements`.

## 5. Дорожня карта по фазах

### Фаза 0 — Каркас (поточна)
- Node.js, проєкт Expo створено, відкривається в Expo Go.
- Встановити: `expo-sqlite`, `expo-image`, `zustand`; налаштувати NativeWind.
- Модуль БД + міграційний раннер (схема v1).
- **Готово, коли:** порожній додаток працює, БД ініціалізується, стилі NativeWind застосовуються.

### Фаза 1 — Логування (ядро)
- Засів вправ із free-exercise-db (+ укр. переклад), прев'ю через expo-image.
- Екран тренування: вибір вправи → запис підходів (вага×повтори) → збереження сесії.
- Активна сесія в Zustand, таймер відпочинку.
- **Готово, коли:** можна провести й зберегти тренування, дані лишаються після перезапуску.

### Фаза 2 — Прогресія
- Міграція **v2** (поля PR/RPE).
- Показ минулих чисел під час логування.
- Оцінка 1ПМ, трекінг персональних рекордів, історія по вправі.
- **Готово, коли:** видно минулі результати, рекорди визначаються автоматично.

### Фаза 3 — Аналітика
- Підрахунок тоннажу, тижневий об'єм по м'язових групах, теплова карта тіла, графіки, календар-стрік (нейтральний).
- **Готово, коли:** графіки будуються з реальних даних.

### Фаза 4 — Експорт і тіло
- Міграція **v3** (вага тіла, заміри).
- Експорт у CSV, трекінг ваги тіла з трендом, фото прогресу.
- **Готово, коли:** CSV-файл вивантажується, є графік динаміки ваги.

### Фаза 5 — Локалізація та сповіщення
- Українська локалізація інтерфейсу + назв вправ, нагадування про тренування.
- **Готово, коли:** додаток повністю українською, нагадування спрацьовують.

## 7. Збірка APK
```
npm install -g eas-cli
eas login
eas build:configure
# у eas.json, профіль preview:  "android": { "buildType": "apk" }
eas build -p android --profile preview
```
Альтернатива без кредитів: `eas build --local` (потрібен Android SDK).
